JDK-1.8 官方文档

1 内存分布

1.1 内存分配策略

  1. 优先分配到Eden
  2. 大对象直接分配到老年代
  3. 长期存活的对象分配到老年代
  4. 空间分配担保
  5. 动态对象年龄判断

1.2 栈上分配

没有逃逸的对象会在栈上分配，而不是在堆分配。JDK-1.8 默认开启逃逸分析。

1.3 查看 JDK 是否开启逃逸分析

2 垃圾回收

2.1 判定对象为垃圾对象

  1. 引用计数法
  2. 可达性分析法

2.2 回收策略

  1. 标记-清除算法
  2. 复制算法（新生代）
  3. 标记-整理算法（老年代）
  4. 分代收集算法

2.3 垃圾回收器

  1. Serial：最基本，发展历史最悠久，单线程，桌面应用
  2. Parnew
  3. Parallel
  4. Cms
  5. G1：并行与并发，分代收集，空间整合，可预测的停顿

2.4 打印垃圾回收的日志

2.5 作为 GC Roots 的对象

  1. 虚拟机栈
  2. 方法区的类属性所引用的对象
  3. 方法区中常量所引用的对象
  4. 本地方法栈中引用的对象

2.6 局部变量表 Slot 复用

当一个变量的 PC 寄存器的值大于 Slot 作用域时，Slot 可以复用。

TIPS：在局部方法中用完对象后，建议置为null，方便GC尽快回收。

2.7 垃圾收集器选型

  1. 若应用程序的数据集较小（小于100MB），选择串行收集器
  2. 若应用程序运行于单核处理器且没有暂停时间的要求，可以交由 JVM 选择收集器或选择串行收集器
  3. 若需优先考虑应用程序峰值性能，没有暂停时间要求或可以接受1秒甚至更长的暂停时间，可以交由 JVM 选择收集器或选择并行收集器
  4. 若应用程序响应时间比整体吞吐量更重要，垃圾回收暂停时间必须短于1秒，则可以选择并发收集器

  1. 使用 new 关键字实例化对象

  3. 调用类的静态方法时
  4. 进行反射调用时
  5. 初始化类时，发现其父类还未初始化时，会先初始化父类

3.3 类不被初始化的情况
