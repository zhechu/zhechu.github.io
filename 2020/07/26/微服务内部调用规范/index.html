微服务内部调用规范

Java 微服务 规范 架构 2020/07/26

目标：规范微服务各层之间调用，提高系统扩展性、可维护性和性能（各层职责分明，引入多线程，甚至可以提高系统吞吐量）

流程 说明
过滤器/拦截器 -> 事件总线 示例：实时拦截非法请求，发邮件/短信通知相关人员
控制器 -> 事件总线 示例：收集用户行为数据（需判断请求是否成功，适合在此层处理），并通过异步持久化到磁盘
服务层 -> 事件总线 示例：解耦同层间调用（同层调用容易混乱，且存在循环依赖风险），支持同步和异步方式
事件总线 -> 消息中间件发送出口 禁止控制层/服务层直接调用消息中间件API，将调用远程API的各种异常统一交给消息中间件发送出口处理（如消息发送降级处理）
消息中间件监听入口 -> 事件总线 可以做消息幂等处理（消息幂等不属于消息中间件职责）
