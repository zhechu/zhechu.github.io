1 分层职责

层 包路径 说明
Controller com.xxx.controller 入参和出参处理，包括校验入参和聚合出参
RPC com.xxx.rpc RPC 调用
Service com.xxx.service 具体业务处理，包括本地事务控制
Dao com.xxx.dao 持久化处理
Mapper com.xxx.dao.mapper 持久化处理，方便扩展，如：复杂查询，需手写 SQL 实现

2 领域模型

领域模型 命名方式 示例 说明
DO（Data Object） <表名+DO> UserDO 此对象与数据库表结构一一对应
BO（Business Object） <表名+BO> UserBO 业务对象，处理对象内部业务逻辑
DVO（Data Validate Object） <方法名+DVO> UserLoginDVO 数据校验对象
DTO（Data Transfer Object） <方法名+DTO> UserUpdateDTO 新增/修改入参对象
QO（Query Object） <方法名+QO> UserQO 查询入参对象（不与 DTO 共用，只服务查询接口）
VO（Response Object） <方法名+VO> UserVO 返回对象

数据校验一般放在聚合层，所以DVO类切勿共享，其只服务于开放出去的接口层（一般为聚合层的Controller层）
